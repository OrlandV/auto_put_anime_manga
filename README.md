### Цели приложения
* Автоматический поиск информации об anime и манге на веб-ресурсах (*requests*, *Wikipedia*).
* Автоматическое добавление найденной информации в базу данных (*mysqlclient*).
* Изменение и сохранение постеров к anime и манге (*Pillow*).

### Задачи приложения (основной алгоритм работы)
1. Ввод данных об искомых anime и манге.
    * На текущем этапе разработки (для упрощения разработки парсера) — поштучное указание в файле `input.py`.
    * На следующем этапе — чтение файлов `CSV` или `JSON`, либо запрос ввода пользователем.
2. На основе вводных данных поиск и сбор страниц (данных) из следующих БД:
    * **World Art** (манга или anime отдельно),
    * **AnimeNewsNetwork** (используя XML API) (отдельно помечаются связи манги и anime),
    * **Wikipedia** (en) (используя библиотеку `Wikipedia`),
    * **MangaUpdates** (используя JSON API).
3. В собранных страницах (данных) поиск ссылок на связанные мангу и anime и сбор соответствующих страниц (данных).

    3.1. **World Art**. Манга из страниц(ы) anime (отдельно помечаются связи манги и anime).

    3.2. **World Art**. Anime из страниц(ы) манги.

    3.3. **World Art** → **AnimeNewsNetwork**. Ссылки из «World Art» в «AnimeNewsNetwork» (отдельно помечаются связи
        манги и anime).

    3.4. **World Art** → **Wikipedia**. Ссылки из «World Art» в «Wikipedia».

    3.5. **Wikipedia**. Выделение из страниц частей инфо-блоков, описывающих отдельную мангу или anime.

    3.6. **Wikipedia**. Если к текущему шагу не нашлось страниц (или отфильтровались), производится повторный поиск, но
        к искомому наименованию добавляется строка ` (manga)`.

    3.7. **World Art** → **MangaUpdates**. Ссылки из «World Art» в «MangaUpdates».

4. Извлечение интересующих данных из страниц.

    4.1. **World Art**. Манга.

    4.2. **World Art**. Anime (с указанием связанной манги).

    4.3. **AnimeNewsNetwork**. Манга.

    4.4. **AnimeNewsNetwork**. Anime (с указанием связанной манги).

    4.5. **Wikipedia**.

    4.6. **MangaUpdates**.

5. Проверка наличия наименований **AnimeNewsNetwork** и **Wikipedia** в **World Art**:
    поиск и добавление отсутствующих страниц. (Почти обратная процедура пунктам **3.3** и **3.4**, после чего новые
    страницы проходят через **3.1** и **3.2**.)
6. Извлечение данных из вновь найденных страниц в **World Art**. (Аналогично пунктам **4.1** и **4.2**.)
7. Формирование единого словаря данных следующего формата:
    ```
    {
        'manga': [
            {
                'wa_id': int,
                'name_orig': str,
                'name_rom': str,
                'name_eng': str,
                'name_rus': str,
                'author_of_manga': [
                    {
                        'name_orig': str,
                        'name_rom': str,
                        'name_rus': str
                    },
                    …
                ],
                'number_of_volumes': int,
                'number_of_chapters': int,
                'date_of_premiere': str,
                'publication': [
                    {
                        'publication': str,
                        'publishing': str,
                        'type': int
                    },
                    …
                ],
                'genre': [
                    str,
                    …
                ],
                'notes': str,
                'poster': str,
                'ann_id': int
            },
            …
        ]
        'anime': [
            {
                'name_orig': str,
                'name_rom': str,
                'name_eng': str,
                'name_rus': str,
                'format': str,
                'number_of_episodes': int,
                'duration': str,
                'date_of_premiere': str,
                'studio': [
                    str,
                    …
                ],
                'director': [
                    {
                        'name_orig': str,
                        'name_rom': str,
                        'name_rus': str
                    },
                    …
                ],
                'genre': [
                    str,
                    …
                ],
                'notes': str,
                'poster': str,
                'manga': int
            },
            …
        ]
    }
    ```
8. Иногда встречаются одноимённые проекты. В таких случаях к наименованию более позднего добавляется год премьеры.
    Это из-за требования БД: уникальность наименований (оригинальных и ромадзи) внутри таблиц манги или anime.
9. Вывод единого словаря данных (таблицы) пользователю и запрос указания добавляемых в БД данных.
10. Проверка наличия в БД наименования (оригинального и ромадзи) **манги**. Если наименование манги в БД отсутствует, то
    перейти к пункту **11.1.** Если наименование манги найдено (получен ID), то перейти к пункту **11.3.**
11. Добавление данных по **манге** в БД.

    11.1. Добавление манги и получение присвоенного ID.

    11.2. Сохранение микропостера манги (из URL) в размере до 100х100 в файле `microposter/m/{ID}.jpg`.

    11.3. Прописывание нового ID манги в связи anime.

    11.4. Перейти к следующей манге (вернуться к пункту **10**).

12. Проверка наличия в БД наименования (оригинального и ромадзи) **anime**. Если наименование anime в БД отсутствует, то
    перейти к пункту **13.1.** Если наименование anime найдено (получен ID), то anime в БД не добавлять, а перейти к
    следующему anime.
13. Добавление данных по **anime** в БД.

    13.1. Добавление anime и получение присвоенного ID.

    13.2. Сохранение микропостера anime (из URL) в размере до 100х100 в файле `microposter/a/{ID}.jpg`.

    13.3. Перейти к следующему anime (вернуться к пункту **12**).

## Использование
1. Отредактировать файл `config.py`.

    1.1. Прописать в словаре `COOKIES_WA` **COOKIE** авторизации на **World Art**.

    1.2. Прописать в словаре `COOKIES_WP` **COOKIE** для обхода анти-бот-защиты **Wikipedia** (en).

    1.3. Прописать в кортеж `DB_CONNECT` параметры подключения к БД ("Сервер", "логин", "пароль", "БД").

    1.4. Прописать путь хранения микропостеров в переменную `PATH`. Например: `'C:/LOCALHOST/anime_manga/microposter/'`.

    1.5. Актуализировать версию браузера Mozilla Firefox для User-Agent (`FIREFOX_VER`).

    1.6. При желании отредактировать следующие словари:<ul>
        <li>`FORM_ANN` — WA-форматы anime в **ANN**,</li>
        <li>`FORM_WP` — WA-форматы anime в **Wikipedia**,</li>
        <li>`IGNORED_GENRES` — игнорируемые жанры,</li>
        <li>`GENRES_ANN` — жанры **AnimeNewsNetwork**,</li>
        <li>`FREQUENCY` — периодичность издания в наименовании.</li>
        </ul>

    1.7. Сохранить изменения файла `config.py`.

2. Отредактировать файл `input.py`.

   2.1. Прописать тип искомого (`'anime'`/`'manga'`) в значение переменной (константы) `A_M`.
       Можно использовать константы `A` и `M`. По умолчанию указана константа `A`.

   2.2. Прописать искомое наименование (на ромадзи) в значение переменной `title`.

   2.3. Прописать год премьеры в значение переменной `year` (вместо `1900`).

   2.4. Прописать формат anime (так, как указано в **World Art**) в значение переменной `form`.
       Если ищется манга (`A_M = M`), то переменная `form` игнорируется — и её можно оставить пустой.

   2.5. Сохранить изменения файла `input.py`.

3. Запустить скрипт `auto_put_anime_manga.py`:
    ```bash
    python auto_put_anime_manga.py
    ```
4. Если появился запрос на добавление нового жанра, согласиться (`Y`) или отказаться (`N`).
    ```text
    Новый жанр в ANN! Adventure
    Добавить жанр? Y/N: 
    ```
    Если согласились, то укажите наименование жанра на русском для добавления в БД.
    ```text
    Наименование жанра на русском: 
    ```
    Если у манги сразу несколько новых жанров, то процедура повторится для каждого.
    После добавления жанра(ов) выведется уведомление.
    ```text
    Перенесите новые жанры в «config.py» из «new_genres.txt».
    ```
5. Если появился запрос на выбор жанров, ввести номера, разделяя пробелом.
    ```text
    Для «{title}» не определены жанры. Выберите жанры из списка и укажите их номера через пробел.
    22. —<Не определён>—
    21. боевик
    11. боевые искусства
    15. детектив
    8. драма
    5. комедия
    13. махо-сёдзё
    3. меха
    16. мистика
    12. музыкальный
    7. повседневность
    1. приключения
    9. романтика
    20. сверхъестественное
    6. спорт
    19. триллер
    17. ужасы
    2. фантастика
    4. фэнтези
    18. школа
    10. эротика
    14. этти
    Ваш выбор:  
    ```
    `{title}` — ромадзи-наименование anime или манги.
6. Если появился запрос на выбор наименования манги, ввести номер.
    ```text
    Выберите {lang} наименование манги «{title}»:
    0. *** Подходящего варианта нет ***
    1. …
    …
    Укажите номер:  
    ```
    `{lang}` — язык наименования манги: `оригинальное`, `ромадзи`, `английское` или `русское`.<br>
    `{title}` — ромадзи-наименование манги.<br>
    Если в списке нет подходящего варианта, тогда нужно указать номер `0`.
7. Проверив итоговые данные, указать добавляемые в БД.
8. Дождаться окончания работы скрипта.
9. Если добавляли новые жанры (шаг `4`), то нужно открыть на редактирование файл `config.py`
    и добавить в словари `GENRES_ANN` и `GENRES_MU` новый(е) жанр(ы) из файла `new_genres.txt` (открыть на чтение).
    После переноса новых жанров содержимое файла `new_genres.txt` можно удалить.
